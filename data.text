2. Explain the difference between COPY INTO and INSERT INTO.

COPY INTO: Used to load bulk data from external files (like CSV, Parquet in OneLake/Blob). Best for ingestion.

INSERT INTO: Used to insert data from another table or query inside Fabric. Best for small-scale inserts.

How do you optimize query performance in Fabric Warehouse?

Use partitioning to scan less data.

Create indexes on frequently used columns.

Avoid SELECT *, fetch only required columns.

Use materialized views or caching for repeated queries.

Ensure data distribution is even to avoid skew.
